# 隐私政策

最后更新：2026 年 6 月 29 日

xlinuxfs 是一个开源的 macOS 工具,用于挂载 Linux 文件系统(ext2/3/4、XFS、Btrfs),使用 Apple 的 FSKit 文件系统扩展模型,并通过 LKL(Linux Kernel Library)接入真正的 Linux 内核作为引擎。

这份隐私政策说明应用和文件系统扩展如何处理数据。简单来说：xlinuxfs 在你的 Mac 本地工作,不投放广告,不追踪用户,也不会把你的文件或设备信息上传到我们的服务器。

## 数据收集

xlinuxfs 不收集个人数据。

应用不包含分析 SDK、广告 SDK、追踪 SDK、用户账号、遥测上报或应用内网络上报功能。

## 本地文件和磁盘访问

xlinuxfs 需要访问 Linux 文件系统卷,才能提供文件系统功能。

当 Linux 磁盘或磁盘映像通过 xlinuxfs 的 FSKit 扩展挂载时,扩展可能会读取和写入该卷上的文件系统数据,包括文件内容、文件名、目录项、文件元数据以及磁盘上的文件系统结构(ext/XFS/Btrfs)。这些处理都是正常文件系统操作的一部分,并且发生在你的 Mac 本地。

xlinuxfs 不会上传你的磁盘内容、文件名、目录列表或磁盘映像。

## 磁盘映像

如果你在应用中选择一个磁盘映像,macOS 会授予应用访问该文件的权限。xlinuxfs 使用这个已选择的路径来显示可复制的附加映像命令。应用不会上传这个映像文件。

## 诊断信息

xlinuxfs 提供诊断界面,用来说明 FSKit 扩展是否已安装、是否已启用,以及是否存在重复注册等问题。

诊断界面可能显示这些本地状态信息：

- 扩展是否已安装
- 扩展是否已启用
- 本机扩展注册路径
- 用于排查问题的终端命令建议

这些信息只在应用内本地显示。xlinuxfs 不会自动把它们发送给我们。

## 剪贴板

由于 macOS 沙盒限制,某些操作会显示为可复制的终端命令,而不是由应用直接执行。当你点击“复制”按钮时,xlinuxfs 会把对应命令写入 macOS 剪贴板。

## 设置

xlinuxfs 会保存简单的本地偏好设置,例如磁盘驱动器和磁盘映像是否默认以只读方式挂载。这些偏好设置通过 macOS 的用户默认设置,保存在一个与文件系统扩展共享的 App Group 容器中,以便扩展在挂载卷时应用它们。

## 网络访问

xlinuxfs 的正常功能不需要网络连接,也不会把应用数据发送到远程服务。

如果你访问项目网站、GitHub 仓库或支持页面,这些服务可能会按照它们自己的隐私政策处理数据。

## 第三方代码

xlinuxfs 使用 Linux 内核(通过 LKL,Linux Kernel Library)及其 ext4/XFS/Btrfs 驱动作为文件系统引擎。这些代码在本地运行,是应用扩展中包含的文件系统实现的一部分,按 GNU 通用公共许可证第 2 版授权。

## 崩溃报告和系统日志

macOS 可能会根据你的系统设置生成崩溃报告或系统日志。xlinuxfs 不会自动接收或上传这些报告。如果你选择在 GitHub issue 或支持请求中分享日志或诊断信息,请先自行检查,并移除你不希望公开的内容。

## 儿童隐私

xlinuxfs 是一个通用工具,不会有意收集儿童数据。
